# 第一部分：总览与架构

## 1. 项目概述

### 1.1 项目名称与定位

- **项目名称**：校园管理系统（Campus Management System）
- **项目定位**：面向高校/中小学校的综合性校园管理 Web 应用，覆盖学生日常申请、教师审批管理、管理员全局管控三大业务场景，并集成 AI 智能助手提升操作效率。
- **用户角色**：学生（Student）、教师（Teacher）、校园管理员（Admin），三角色各自拥有独立的功能模块和权限边界。

### 1.2 技术栈说明

| 层级 | 技术选型 | 版本 | 选型理由 |
|------|---------|------|---------|
| 前端框架 | React | 18.x | 生态成熟，组件化开发效率高 |
| 类型系统 | TypeScript | 5.x | 类型安全，大型项目可维护性强 |
| UI 组件库 | Ant Design | 5.x | 企业级中后台首选，表格/表单/日历等内置组件丰富 |
| 状态管理 | Zustand | 4.x | 轻量、无模板代码，比 Redux 更适合中型应用 |
| HTTP 客户端 | axios | 1.x | 拦截器机制适合统一处理认证/错误 |
| 后端框架 | NestJS | 10.x | 企业级 Node.js 框架，模块化 + 装饰器 + 依赖注入 |
| 数据库 | MySQL | 8.0+ | 关系型数据库，适合校园管理这类强 schema 场景 |
| ORM | TypeORM | 0.3.x | 与 NestJS 深度集成，支持 Migration |
| 认证 | JWT (jsonwebtoken) | — | 无状态认证，适合前后端分离架构 |
| 文件存储 | OSS 抽象层 | — | 支持阿里云 OSS / AWS S3，通过适配器模式切换 |
| AI 对接 | Claude / DeepSeek API | — | SSE 流式响应，长文本理解能力强 |
| 实时通信 | SSE (Server-Sent Events) | — | AI 对话流式输出；通知推送走轮询 + SSE 混合 |

### 1.4 设计原则与约定

| 原则 | 说明 |
|------|------|
| **RESTful 风格** | API 路径用名词复数，HTTP 方法语义化（GET 查询/POST 创建/PUT 更新/DELETE 删除） |
| **统一响应格式** | 所有接口返回 `{ code, message, data }` 标准结构 |
| **分层架构** | 后端严格分层：Controller → Service → Repository → Entity |
| **前端组件化** | 页面 = 布局组件 + 业务组件 + 通用组件，复用优先 |
| **类型安全** | 前后端共享 DTO 类型定义（接口请求/响应类型） |
| **权限三明治** | 接口级守卫（NestJS Guard）+ 页面级路由守卫（React Router）+ 组件级条件渲染 |
| **手机号即账号** | 学生/教师以手机号登录，管理员为系统预置账号 |
| **软删除** | 核心业务数据标记 `deleted_at` 字段，不做物理删除 |

### 2.2 登录认证方案

```
┌──────────┐         ┌──────────────┐         ┌──────────┐
│  前端     │ ①POST   │  后端 Auth   │ ②查询   │  MySQL   │
│  登录页   │ ──────▶ │  Controller  │ ──────▶ │  users   │
│          │         │              │         │          │
│          │ ④返回   │  ③bcrypt     │         │          │
│          │ ◀────── │  验证密码    │         │          │
│          │  JWT    │              │         │          │
│          │         │  签发 JWT    │         │          │
└──────────┘         └──────────────┘         └──────────┘
     │
     │ ⑤后续请求 Header: Authorization Bearer <token>
     ▼
┌──────────┐         ┌──────────────┐
│  前端     │         │  JWT Guard   │ → 解析 token → 注入 req.user
│  axios    │ ──────▶ │  角色 Guard  │ → 读取用户角色 → 放行/拦截
│  拦截器   │         │              │
└──────────┘         └──────────────┘
```

**认证核心参数**：

| 配置项 | 值 | 说明 |
|--------|-----|------|
| Token 类型 | JWT (HS256) | 对称加密，服务端持有 secret |
| Access Token 有效期 | 2 小时 | 短期，降低泄露风险 |
| Refresh Token 有效期 | 7 天 | 长期，用于无感刷新 |
| 刷新策略 | 静默刷新 | 前端 axios 响应拦截器检测 401 → 用 refresh token 换新 access token |
| 密码加密 | bcrypt (saltRounds=10) | 不可逆哈希 |
| 登录方式 | 手机号 + 密码 | 学生/教师用手机号；管理员用用户名 |

```

## 3. 数据库设计

### 3.1 实体关系图（ER 图文字描述）

```
users (用户基础表)
  ├── 1:1 ── students (学生扩展信息)
  ├── 1:1 ── teachers (教师扩展信息)
  └── 1:N ── notifications (通知消息)

  └── 1:N ── leave_requests (请假申请)

  └── 1:N ── leave_approvals (审批记录)

## 4. API 接口设计

### 4.1 通用规范

#### 4.1.1 基础 URL

```
开发环境：http://localhost:3000/api
生产环境：https://<domain>/api
```

#### 4.1.2 统一响应格式

```json
{
  "code": 0,
  "message": "success",
  "data": { },
  "timestamp": 1718500000000
}
```

| code | 含义 |
|------|------|
| 0 | 成功 |
| 400 | 请求参数错误 |
| 401 | 未认证（Token 缺失/过期） |
| 403 | 无权限（角色不匹配） |
| 404 | 资源不存在 |
| 409 | 资源冲突（如时间段已被预约） |
| 500 | 服务器内部错误 |

#### 4.1.3 分页响应格式

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "items": [],
    "pagination": {
      "page": 1,
      "pageSize": 20,
      "total": 150,
      "totalPages": 8
    }
  }
}
```

### 5.3 中间件清单

| 中间件 | 作用 | 作用范围 |
|--------|------|---------|
| `JwtAuthGuard` | 全局认证守卫，解析 JWT 并注入 `req.user` | 全局（除 @Public 路由） |
| `RolesGuard` | 读取路由 @Roles 装饰器元数据，比对用户角色 | 需要角色限制的路由 |
| `ValidationPipe` | 自动校验请求体是否符合 DTO 的 class-validator 装饰器规则 | 全局 |
| `ResponseInterceptor` | 将 Controller 返回值统一封装为 `{ code:0, message:'success', data }` | 全局 |
| `HttpExceptionFilter` | 捕获所有未处理的异常，格式化为统一错误响应 | 全局 |
| `LoggingInterceptor` | 记录每个请求的方法/路径/耗时/响应状态码 | 全局 |
| `CorsMiddleware` | 处理跨域请求 | 全局 |

// 每个页面的标准结构
const PageTemplate: React.FC = () => {
  // 1. 状态
  const [data, setData] = useState<ItemType[]>([]);
  const [loading, setLoading] = useState(true);
  const [pagination, setPagination] = useState({ page: 1, pageSize: 20, total: 0 });

  // 2. 数据获取
  const fetchData = useCallback(async (params: QueryParams) => {
    setLoading(true);
    try {
      const res = await api.getList(params);
      setData(res.data.items);
      setPagination(res.data.pagination);
    } catch (error) {
      message.error('加载失败');
    } finally {
      setLoading(false);
    }
  }, []);

  // 3. 搜索区
  const searchBar = (
    <Form layout="inline" onFinish={handleSearch}>
      {/* 搜索字段 */}
    </Form>
  );

  // 4. 工具栏
  const toolbar = (
    <Space>
      <Button type="primary" icon={<PlusOutlined />} onClick={handleCreate}>
        新建申请
      </Button>
    </Space>
  );

  // 5. 表格
  const table = (
    <Table
      rowKey="id"
      columns={columns}
      dataSource={data}
      loading={loading}
      pagination={{
        ...pagination,
        onChange: (page, pageSize) => fetchData({ page, pageSize }),
        showSizeChanger: true,
        showTotal: (total) => `共 ${total} 条`,
      }}
    />
  );

  // 6. 模态框（创建/编辑/详情）
  const modal = (
    <Modal open={modalOpen} onCancel={closeModal} onOk={handleSubmit}>
      {/* 表单内容 */}
    </Modal>
  );

  // 7. 渲染
  return (
    <PageContainer>
      {searchBar}
      <br />
      {toolbar}
      <br />
      {table}
      {modal}
    </PageContainer>
  );
};

### 7.3 路由守卫实现

```typescript
// RequireAuth.tsx — 认证守卫
const RequireAuth: React.FC<{ children: React.ReactNode }> = ({ children }) => {
  const { user, isAuthenticated } = useAuthStore();

  if (!isAuthenticated) {
    return <Navigate to="/login" replace />;
  }
  return <>{children}</>;
};

// RequireRole.tsx — 角色守卫
const RequireRole: React.FC<{ allowed: Role[]; children: React.ReactNode }> = ({
  allowed,
  children,
}) => {
  const { user } = useAuthStore();

  if (!user || !allowed.includes(user.role)) {
    return <Navigate to="/403" replace />;
  }
  return <>{children}</>;
};
```

### 7.4 懒加载策略

- 所有页面组件使用 `React.lazy(() => import('./pages/...'))` 实现代码分割
- 加载态统一使用 `<Spin size="large" />` 全屏居中
- 登录页不懒加载（首屏关键路径）

### 11.2 API 请求封装

```typescript
// services/http.ts — axios 实例 + 拦截器
const http = axios.create({
  baseURL: import.meta.env.VITE_API_BASE_URL || '/api',
  timeout: 15000,
  headers: { 'Content-Type': 'application/json' },
});

// 请求拦截器 — 自动注入 Token
http.interceptors.request.use((config) => {
  const token = useAuthStore.getState().accessToken;
  if (token) {
    config.headers.Authorization = `Bearer ${token}`;
  }
  return config;
});

// 响应拦截器 — 统一错误处理 + Token 刷新
http.interceptors.response.use(
  (response) => {
    const { code, message: msg } = response.data;
    if (code !== 0) {
      antMessage.error(msg || '请求失败');
      return Promise.reject(new Error(msg));
    }
    return response.data; // 直接返回 { code, message, data }
  },
  async (error) => {
    if (error.response?.status === 401) {
      // 尝试刷新 Token
      const refreshed = await useAuthStore.getState().refreshAccessToken();
      if (refreshed) {
        return http(error.config); // 重试原请求
      }
      // 刷新失败 → 踢到登录页
      useAuthStore.getState().logout();
      window.location.href = '/login';
    }
    return Promise.reject(error);
  }
);
```

**前端 SSE 消费**（`services/ai.service.ts`）：

```typescript
async function chatWithAI(
  message: string,
  sessionId: string,
  context: AIContext,
  onToken: (text: string) => void,
  onAction: (action: AIAction) => void,
  onDone: (sessionId: string) => void,
  onError: (error: Error) => void,
): Promise<void> {
  const response = await fetch('/api/ai/chat', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      Authorization: `Bearer ${getToken()}`,
    },
    body: JSON.stringify({ message, sessionId, context }),
  });

  const reader = response.body!.getReader();
  const decoder = new TextDecoder();
  let buffer = '';

  while (true) {
    const { done, value } = await reader.read();
    if (done) break;

    buffer += decoder.decode(value, { stream: true });
    const lines = buffer.split('\n');
    buffer = lines.pop() || '';

    for (const line of lines) {
      if (line.startsWith('data: ')) {
        const data = JSON.parse(line.slice(6));
        if (data.type === 'text') onToken(data.content);
        if (data.type === 'action') onAction(data);
        if (data.type === 'done') onDone(data.sessionId);
      }
    }
  }
}
```

#### 请假表单校验

| 字段 | 规则 |
|------|------|
| leaveType | 必选，ENUM 值 |
| startDate / endDate | 必选，startDate ≤ endDate，不能选择过去日期 |
| startPeriod / endPeriod | 必选，startPeriod ≤ endPeriod，1-10 范围内 |
| reason | 必填，最少 5 个字符，最长 500 个字符 |
| attachmentUrls | 可选，最多 3 张，单张 ≤ 5MB，支持 jpg/png |

#### 场地预约表单校验

| 字段 | 规则 |
|------|------|
| bookingDate | 必选，今天~今天+max_days_ahead 范围内 |
| startPeriod / endPeriod | 必选，startPeriod ≤ endPeriod，不超过单日最大预约节数 |
| purpose | 必填，最少 2 个字符，最长 200 个字符 |
| attendeeCount | 必填，正整数，≤ 场地 capacity |

#### 报修表单校验

| 字段 | 规则 |
|------|------|
| title | 必填，最少 2 个字符，最长 50 个字符 |
| category | 必选，ENUM 值 |
| urgency | 必选，ENUM 值 |
| location | 必填，最少 2 个字符，最长 100 个字符 |
| description | 必填，最少 10 个字符，最长 1000 个字符 |
| imageUrls | 可选，最多 5 张，单张 ≤ 10MB |

### 14.4 后端异常处理策略

| 异常类型 | HTTP 状态码 | 处理方式 |
|---------|-----------|---------|
| 参数校验失败（class-validator） | 400 | 返回具体字段错误信息 |
| JWT 无效/过期 | 401 | 返回 `{ code: 401, message: '认证失败' }` |
| 角色权限不足 | 403 | 返回 `{ code: 403, message: '无权限' }` |
| 资源不存在 | 404 | 返回 `{ code: 404, message: '资源不存在' }` |
| 时间段冲突（唯一约束） | 409 | 返回 `{ code: 409, message: '该时间段已被预约' }` |
| 文件上传失败（OSS） | 500 | 记录日志，返回 `{ code: 500, message: '文件上传失败' }` |
| 数据库连接失败 | 500 | 记录日志，返回通用错误信息（不泄露内部细节） |
| AI API 超时/不可用 | 502 | 返回 `{ code: 502, message: 'AI 服务暂时不可用' }` |
| 未捕获异常 | 500 | `HttpExceptionFilter` 统一捕获，记录堆栈，返回通用错误 |
